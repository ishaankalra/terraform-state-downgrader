# Sample Terraform Project with Modules

This directory contains a sample Terraform configuration used for testing the new reliable provider parsing approach.

## Structure

```
.
├── main.tf                      # Root configuration
├── modules/
│   ├── database/
│   │   └── main.tf             # Database module
│   └── network/
│       ├── main.tf             # Network module
│       └── subnet/
│           └── main.tf         # Nested subnet module
└── .terraform/
    ├── modules/
    │   └── modules.json        # Module metadata (generated by terraform init)
    └── providers/              # Downloaded providers
```

## New Reliable Parsing Approach

Instead of parsing the fragile `terraform providers` tree output, we now use:

### 1. Parse `modules.json`
```json
{
  "Modules": [
    {"Key": "", "Source": "", "Dir": "."},
    {"Key": "database", "Source": "./modules/database", "Dir": "modules/database"},
    {"Key": "network", "Source": "./modules/network", "Dir": "modules/network"},
    {"Key": "network.subnet", "Source": "./subnet", "Dir": "modules/network/subnet"}
  ]
}
```

### 2. Parse `.tf` files directly
For each module directory, parse `required_providers` blocks:

```hcl
terraform {
  required_providers {
    random = {
      source  = "hashicorp/random"
      version = "~> 3.5.1"
    }
  }
}
```

Extracted data:
- Name: `random`
- Source: `hashicorp/random`
- Version: `~> 3.5.1`
- Module: `database`

### 3. Map to `.terraform.lock.hcl`
Use the lock file to resolve source to FQDN:
```
hashicorp/random → registry.terraform.io/hashicorp/random
```

## Benefits

✅ **Reliable** - Direct source of truth (HCL files)
✅ **No tree parsing** - No fragile regex on `terraform providers` output
✅ **Module-aware** - Tracks which module requires which provider
✅ **Works without modules** - Gracefully handles projects without `modules.json`
✅ **Works with all configs** - Any valid Terraform configuration
